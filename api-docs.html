Introduction

The PyToolbox API provides programmatic access to PyToolbox's features, allowing you to integrate our platform into your applications, websites, or workflows.

Base URL

https://api.pytoolbox.xyz/v1

Response Format

All API responses are returned in JSON format. Each response includes a status field indicating whether the request was successful, and a data field containing the requested information.

{
  "status": "success",
  "data": {
    // Response data here
  }
}

For error responses, the structure includes an error object with details about what went wrong:

{
  "status": "error",
  "error": {
    "code": "invalid_auth",
    "message": "Invalid API key provided",
    "details": "The API key you provided was not recognized"
  }
}

Authentication

To access the PyToolbox API, you need an API key. You can obtain one by registering for a PyToolbox account and generating an API key in your dashboard.

API Key Authentication

Include your API key in the request headers:

curl -X GET "https://api.pytoolbox.xyz/v1/tools" \
  -H "X-API-Key: your_api_key_here"

Important

Keep your API key secure and never expose it in client-side code. If you believe your API key has been compromised, you can regenerate it from your dashboard.

Authentication Scopes

When generating an API key, you can specify which scopes (permissions) it should have:

Scope Description
read:tools Read-only access to tools data
read:tutorials Read-only access to tutorials
write:favorites Ability to add/remove favorites
read:user Access to user profile information
write:user Ability to update user profile

Rate Limits

To ensure fair usage and system stability, the PyToolbox API implements rate limiting.

Limits

Plan Rate Limit
Free 100 requests per hour
Pro 1,000 requests per hour
Enterprise 10,000 requests per hour

Rate Limit Headers

Each API response includes headers that provide information about your current rate limit status:

Header Description
X-RateLimit-Limit The maximum number of requests you're permitted to make per hour
X-RateLimit-Remaining The number of requests remaining in the current rate limit window
X-RateLimit-Reset The time at which the current rate limit window resets, in UTC epoch seconds

Exceeding Rate Limits

If you exceed your rate limit, you'll receive a 429 Too Many Requests response with a JSON error message:

{
  "status": "error",
  "error": {
    "code": "rate_limit_exceeded",
    "message": "Rate limit exceeded",
    "details": "You have exceeded your rate limit. Please try again after 23 minutes."
  }
}

Tools Endpoints

The Tools API allows you to retrieve information about Python tools available in the PyToolbox ecosystem.

GET /tools List All Tools

Returns a paginated list of all tools.

Query Parameters

Parameter Type Default Description
page integer 1 Page number for pagination
limit integer 20 Number of tools per page (max 100)
category string null Filter tools by category
sort string popularity Sort by: popularity, name, newest

Example Request

curl -X GET "https://api.pytoolbox.xyz/v1/tools?category=data-science&limit=5" \
  -H "X-API-Key: your_api_key_here"

Example Response

{
  "status": "success",
  "data": {
    "tools": [
      {
        "id": "pandas",
        "name": "Pandas",
        "description": "Powerful data analysis and manipulation library",
        "category": "data-science",
        "version": "1.4.2",
        "website": "https://pandas.pydata.org/",
        "github": "https://github.com/pandas-dev/pandas",
        "popularity": 4.8,
        "tags": ["data-analysis", "dataframes", "time-series"]
      },
      // More tools...
    ],
    "pagination": {
      "total": 45,
      "page": 1,
      "limit": 5,
      "pages": 9
    }
  }
}
GET /tools/{tool_id} Get Tool Details

Returns detailed information about a specific tool.

Path Parameters

Parameter Type Description
tool_id string The unique identifier of the tool

Example Request

curl -X GET "https://api.pytoolbox.xyz/v1/tools/pandas" \
  -H "X-API-Key: your_api_key_here"
